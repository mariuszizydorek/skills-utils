Private Repo Auth + CLI Diagnostics

Fix skills-sync failures fast

Use this guide when `skills-sync` cannot access repositories, especially private GitHub repos. The checks below are ordered for the fastest path to a working setup.

Start command Run `skills-sync doctor` first.
Auth order `--token` -> env token -> `gh auth token`.
Private repos Require read access + valid auth.

Quick health check

Run the doctor command to validate config, auth, GitHub API reachability, and repo access.

CLI 
skills-sync doctor

If you use the Node package command name:

Node CLI 
skills-sync-node doctor

Private repository access failures

If sync fails for a private repo, follow this sequence.

1) Confirm account access

Check that your account/token can read the repository:

GitHub API probe 
gh api repos/<owner>/<repo>
If this returns 404 or 403, request repo access first.

2) Confirm GitHub auth on your machine

Auth status 
gh auth status

If needed:

Login 
gh auth login

3) Use explicit token auth (recommended in CI)

Token-based run 
export GITHUB_TOKEN=ghp_xxx
skills-sync --token "$GITHUB_TOKEN"

Auth resolution order

`skills-sync` resolves GitHub auth in this order:

  1. --token
  2. GITHUB_TOKEN or GH_TOKEN
  3. gh auth token
Use `--token` for deterministic behavior in CI and scripted runs.

Token permission notes

  • Fine-grained PATs need read access to the target repository.
  • Classic PATs should include repo read scope for private repositories.

Common symptoms and fixes

"No GitHub auth detected"

Fix one of:

  • run gh auth login
  • export GITHUB_TOKEN
  • pass --token explicitly

"GitHub API unreachable"

Check network and GitHub status, then retry:

Connectivity check 
curl -I https://api.github.com

Repo listed in config but "unreachable"

  • Verify owner/repo is correct in config.
  • Confirm default branch exists and repo is not archived or renamed.
  • Re-run skills-sync doctor after fixing auth or config.
